# 界面与配色规范

## 设计目标

- 保持 macOS 工具类应用的清爽和层次感，不做厚重的“网页后台感”。
- 设置窗口使用“顶部分类导航 + 卡片式内容”，减少单页堆叠带来的压迫感。
- 说明性文字尽量改为 hover 提示，主界面只保留决策必需的信息。
- 顶部悬浮提示和设置窗口共用同一套视觉 token，避免风格割裂。

## 参考方向

- Apple Human Interface Guidelines
  - https://developer.apple.com/design/human-interface-guidelines/
- Radix Colors
  - https://www.radix-ui.com/colors

本项目当前的视觉方向是：

- 基底使用冷中性色，参考 Apple HIG 对 layered materials 和内容层级的处理。
- 强调色使用接近 `cyan / sky` 的青蓝色，保证“工具状态”和“焦点反馈”清晰。
- 辅助提示色使用琥珀色，只用于 hover 提示图标和轻警示信息，不抢主视觉。

## 配色 Token

代码来源：

- `Caps Nav/Support/CapsNavTheme.swift`

## App 图标

- App 图标沿用窗口主题的冷色体系，主背景使用深蓝到青蓝的渐变。
- 视觉主体为“键帽 + Caps Lock 象征 + 导航十字”的组合，强调它是一个前缀导航工具，而不是通用输入法工具。
- 高光层使用轻微的青蓝与暖琥珀反射，目的是让图标在 Dock 和 Finder 中更有辨识度，但不走高饱和卡通风。
- 生成脚本放在 `tools/GenerateAppIcon.swift`，后续如果改品牌色，优先更新脚本里的颜色常量和本文档，而不是手工逐张改 PNG。

## 使用规则

### 设置窗口

- 页面背景只使用 `windowTop`、`windowBottom` 和两层 glow，不额外叠加高饱和纯色背景。
- 主卡片统一使用 `surfacePrimary`，并配合 `borderSoft` 表达层级。
- 选中的顶部分类导航使用 `accentSoft`，不要直接大面积铺 `accentStrong`。
- hover 提示图标统一使用 `warning`，表示“这里有补充信息”，而不是错误。
- 主题模式支持 `随系统 / 亮色 / 暗色`。新增页面或窗口时，要保证三种模式下的对比度和层级都成立，而不是只在亮色下检查。

### 文本与信息层级

- 页面标题使用 `textPrimary`。
- 说明、辅助文本、路径和次级状态使用 `textSecondary`。
- 不关键的标签或统计信息使用 `textMuted`。
- 错误与风险提示只在必要时使用 `danger`。

### 交互反馈

- 焦点输入框边框使用 `accentStrong`。
- 状态成功、已启用、运行中的强调，优先使用 `accentStrong` 或 `success`。
- 提示关闭、缺权限、需要关注的状态使用 `warning`，不要直接用红色。

### 悬浮帮助框

- 默认位置为 `右侧`。
- 空态保持轻量尺寸；按住 Caps 且有帮助内容时，宽度和列数按屏幕尺寸、可视区域和悬浮位置自适应。
- 顶部、左侧、右侧都优先双列展示；仅在顶部窄屏时允许降为单列。
- 悬浮帮助框不依赖内部滚动；需要优先通过宽度、头部密度和整体高度控制来保证内容一次展示完整。
- 背景使用 `surfacePrimary -> surfaceSecondary` 渐变，不使用纯黑透明面板。
- 支持调节整体透明度，默认 `82%`，范围 `20% - 100%`；文字、背景、边框和状态徽标同步变化。
- 键位标签使用 `accentSoft` 胶囊底，动作说明使用 `textSecondary`。

## 后续约束

- 新增设置页面时，优先复用 `CapsNavTheme`，不要直接在视图里写裸 `hex`。
- 新增说明文案时，默认先考虑 hover 提示，而不是直接堆叠正文段落。
- 如果后续加入更多状态色，先补到 `CapsNavTheme.swift` 和本文档，再落到页面代码。

## 安装盘 DMG

- 安装盘不是默认 Finder 文件夹视图，而是带品牌背景图的视觉化安装页。
- 视觉方向保持和 App 图标一致：深蓝基底、青蓝高光、少量暖琥珀反射。
- 背景图左侧放品牌主视觉，右侧保留安装操作区，避免图标与装饰元素混在一起。
- 右侧安装区的图标与文件名落点，需要有更亮的底板承托，避免 Finder 默认黑色文件名与深色背景混在一起。
- 安装文案只保留一句：`拖动到 Applications 即可安装`。
- DMG 背景图生成脚本放在 `tools/GenerateDMGBackground.swift`，后续如果改品牌风格，优先更新脚本和本文档，不要直接手工改最终 PNG。
