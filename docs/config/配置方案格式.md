# 配置方案格式

Caps Nav 使用“一个配置方案（profile）对应一份配置文件（JSON 文件）”的方式存储按键映射。

默认存储位置：

```text
~/Library/Application Support/Caps Nav/profiles/
```

## 顶层结构

一个最基本的配置方案长这样：

```json
{
  "id": "default",
  "name": "默认方案",
  "version": 2,
  "mappings": []
}
```

字段说明：

- `id`
  - 配置方案唯一标识
- `name`
  - 用户可见名称
- `version`
  - 当前仓库默认版本为 `2`
- `mappings`
  - 当前方案里的全部按键映射

## 一条映射的完整示例

```json
{
  "trigger": {
    "key": "f",
    "modifiers": []
  },
  "output": {
    "type": "builtin",
    "action": "moveRight"
  },
  "description": "光标向右移动"
}
```

## `trigger`

`trigger` 描述的是“按住 `Caps Lock` 后，用户接下来按什么键会命中这条规则”。

示例：

```json
{
  "key": "s",
  "modifiers": ["shift"]
}
```

字段说明：

- `key`
  - 前缀后的触发键
  - 当前实现以字母、符号和常见按键名为主
- `modifiers`
  - 额外修饰键
  - 只允许：`shift`、`control`、`option`、`command`

注意：

- `Caps Lock` 是固定前缀，不写进配置文件
- 不允许定义 `prefix` 字段

## `output`

`output` 描述的是“这条规则命中后，Caps Nav 应该做什么”。

当前支持两种类型：

### `builtin`

```json
{
  "type": "builtin",
  "action": "moveLeft"
}
```

表示：

- 使用 Caps Nav 内置动作
- 例如移动、选择、删除、到行首或行尾

### `shortcut`

```json
{
  "type": "shortcut",
  "shortcut": {
    "key": "a",
    "modifiers": ["control"]
  }
}
```

表示：

- 不走内置语义动作
- 而是直接向系统重发一组自定义快捷键

## `description`

`description` 是用户可见说明文案。

示例：

```json
"description": "光标向右移动"
```

它主要会出现在：

- 设置页中的快捷键说明
- 悬浮帮助提示
- 配置方案预览

规则：

- 允许为空或缺失
- 如果缺失，界面会根据动作类型回退生成说明
- 建议始终写清楚，方便用户理解和维护

## 校验规则

当前实现会检查这些约束：

- `profile.id` 不能为空
- `profile.name` 不能为空
- `trigger.key` 不能为空
- 同一配置方案内，`trigger.key + trigger.modifiers` 不能重复
- `builtin` 不能携带 `shortcut`
- `shortcut` 不能携带 `action`
- `description` 允许为空或缺失；保存时会去掉纯空白字符串

## 默认方案相关规则

- 首次启动会从仓库内置默认配置生成默认方案
- 默认方案名称是 `默认方案`
- 默认方案在设置页中是只读的
- 推荐复制默认方案后再做个性化修改

仓库中的默认配置来源：

- `Caps Nav/Resources/default-profile.json`
- `default_shortcut_description.md`

## 当前代码入口

- 模型：`Caps Nav/Models/Profile.swift`
- 读取与校验：`Caps Nav/Infrastructure/ProfileLoader.swift`
- 方案管理：`Caps Nav/Infrastructure/ProfileManager.swift`
