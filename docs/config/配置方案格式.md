# 配置方案格式

## 顶层结构

```json
{
  "id": "default",
  "name": "默认方案",
  "version": 1,
  "mappings": []
}
```

## trigger

```json
{
  "key": "s",
  "modifiers": ["shift"]
}
```

- `key`：触发键，当前实现以字母/常见键名为主。
- `modifiers`：只允许 `shift`、`control`、`option`、`command`。
- 不允许定义 `prefix` 字段。

## output

### builtin

```json
{
  "type": "builtin",
  "action": "moveLeft"
}
```

### shortcut

```json
{
  "type": "shortcut",
  "shortcut": {
    "key": "a",
    "modifiers": ["control"]
  }
}
```

## mapping.description

```json
{
  "trigger": {
    "key": "f",
    "modifiers": []
  },
  "output": {
    "type": "builtin",
    "action": "moveRight"
  },
  "description": "右移一个字符"
}
```

- `description` 为可选字段。
- 如果存在，该字段会作为设置页和悬浮帮助框中的优先说明文案。
- 如果缺失，设置页会显示“添加说明”入口，悬浮帮助框会回退到内置动作建议文案。

## 校验规则

- `profile.id` 不能为空。
- `profile.name` 不能为空。
- `trigger.key` 不能为空。
- 同一 profile 内 `trigger.key + trigger.modifiers` 不能重复。
- `builtin` 不能携带 `shortcut` 字段。
- `shortcut` 不能携带 `action` 字段。
- `description` 允许为空或缺失；保存时会自动去掉纯空白字符串。
