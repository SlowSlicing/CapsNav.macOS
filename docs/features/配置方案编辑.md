# 配置方案编辑

## 功能目标

让用户在设置页内完成配置方案切换与映射维护，同时保证默认方案始终作为稳定基线保留。

## 用户入口

- 菜单栏 `打开设置`
- 设置页 `配置方案`

## 代码入口

- `Caps Nav/Features/Preferences/PreferencesRootView.swift`
- `Caps Nav/App/AppBootstrap.swift`

## 当前能力

- 横向展示全部配置方案卡片，支持切换当前激活方案。
- 支持新增配置方案，默认从仓库内置默认方案复制一份出来。
- 支持复制、重命名、删除自定义配置方案。
- 支持拖拽排序，排序结果会立即持久化到 `settings.json`。
- 支持在 Finder 中定位单个配置方案 JSON 文件。
- 支持图形化维护按键映射：触发键、功能、删除、新增。
- 内置动作支持“功能效果”列的悬停动态预览，并支持自动播放与手动切换。
- 自定义快捷键支持 `shortcut` 输出，悬停后展示快捷键说明。

## 默认方案规则

- `默认方案` 不能重命名。
- `默认方案` 不能删除。
- 当激活的是 `默认方案` 时，按键映射列表进入只读状态。
- 只读状态下不能新增、修改或删除任何映射。

WHY：
- 默认方案既是首次启动的稳定基线，也是文档、README、默认 JSON 和代码内兜底行为的共同来源。
- 如果允许直接改默认方案，后续很容易出现“用户看到的默认行为”和仓库文档不一致的问题。

## 按键映射编辑规则

- `快捷键` 列负责选择前缀后的触发键。
- `功能` 列支持两类输出：
  - `builtin`：软件内置动作
  - `shortcut`：用户自定义快捷键
- `快捷键说明` 列跟随当前输出自动生成描述，不单独手写。
- 同一配置方案内，触发键不能重复；已被占用的键会在菜单中置灰。

## 功能效果列

### builtin

- 使用与“快捷键练习”同一套文本沙盘语义。
- 列表里只显示一个预览小图标。
- 鼠标悬停后，通过浮层循环演示文本效果，让用户直接看到光标、选区或文本变化。
- 浮层内支持手动切换“动作前 / 动作后”，也可以暂停或恢复自动播放。

### shortcut

- 不展示统一文本效果动画。
- 悬停浮层只展示会发送的快捷键说明。
- 原因是最终效果由目标应用对该快捷键的定义决定，Caps Nav 无法替应用承诺结果。

## 持久化

- 每个配置方案对应一份 JSON 文件。
- 默认目录：
  - `~/Library/Application Support/Caps Nav/profiles/`
- 排序信息和当前激活方案保存在：
  - `~/Library/Application Support/Caps Nav/settings.json`

## 验证重点

- 激活默认方案后，映射列表不能新增、修改、删除。
- 复制默认方案后，新方案应立即可编辑。
- 选择内置动作后，“功能效果”列应显示预览图标，悬停可看到循环动画。
- 选择自定义快捷键后，“功能效果”列应显示说明图标，悬停可看到快捷键说明。
- 拖拽排序、重命名、删除方案后，设置页与本地 JSON 状态保持一致。
