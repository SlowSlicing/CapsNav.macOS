# 自定义快捷键

## 功能目标

让用户把某个 `trigger` 映射成任意目标快捷键，而不是只能使用 Caps Nav 的内置语义动作。

适用场景例如：

- `Caps + q` 转发成 `Control + A`
- `Caps + h` 转发成 `Command + Left`
- `Caps + n` 转发成 `Command + Shift + Right`

## 配置格式

在配置方案文件（也就是 profile JSON 文件）中，使用 `output.type = "shortcut"` 表示这是一条自定义快捷键输出：

```json
{
  "type": "shortcut",
  "shortcut": {
    "key": "a",
    "modifiers": ["control"]
  }
}
```

字段含义：

- `type`
  - 固定写 `shortcut`
- `shortcut.key`
  - 目标快捷键的主键
- `shortcut.modifiers`
  - 目标快捷键附带的修饰键数组

## 当前 UI 交互

设置页的“配置方案 -> 按键映射”里，`功能` 列现在支持两种选择方式：

- 直接选择内置动作 `builtin`
- 打开“自定义快捷键”面板，配置 `shortcut`

自定义快捷键面板当前支持：

- 默认使用“直接录入”，点击录入框后可直接按下目标快捷键
- 也可以切换到“列表选择”，手动选择主键与修饰键
- 勾选 `Shift / Control / Option / Command`
- 实时预览最终发送的快捷键
- 保存后立即写回当前配置方案文件

保存后：

- `output.type` 会变成 `shortcut`
- `description` 会自动同步为“发送 XXX”
- 设置页说明列和悬浮提示会显示同一份文案

## 当前支持范围

目标快捷键主键当前支持：

- 字母
- 数字
- 常用符号
- 方向键
- `Delete`
- `Forward Delete`
- `Return`
- `Space`
- `Tab`
- `Escape`

修饰键当前支持：

- `shift`
- `control`
- `option`
- `command`

## 当前限制

- 更复杂的媒体键、功能键、布局相关按键后续再补。
