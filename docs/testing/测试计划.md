# 测试计划

## 第一阶段目标

验证“菜单栏应用 + 权限 + 默认 profile + 事件拦截 + 合成按键”主链路可用。

## 手工测试清单

### 启动与存储

- 首次启动后生成 `~/Library/Application Support/Caps Nav/`
- 自动生成 `profiles/default.json`
- 自动生成 `settings.json`
- 重启后能恢复 active profile
- 无论是否已授权，启动后都只显示状态栏图标，不自动打开设置页
- 未授权时可通过状态栏入口或设置页内按钮主动打开权限说明窗口

### 权限

- 未授权时设置页能看到未授权状态
- 点击授权按钮后能跳转系统辅助功能面板
- 授权后能自动刷新为已授权
- “检查权限”按钮手动点击后也能更新为已授权

### 菜单栏

- 启动后只显示菜单栏图标，不显示 Dock 常驻图标
- 菜单栏可打开设置页
- 菜单栏可打开快捷键练习
- 菜单栏可切换 active profile
- 菜单栏可直接启用/暂停 Caps Nav，并与概览页开关保持同步
- 暂停状态下，状态栏图标会显示明显的暂停态标识
- 切换状态栏图标样式后，菜单栏图标会立即刷新

### 键盘链路

- 关闭运行总开关后，`Caps Lock` 恢复系统原生行为
- 关闭运行总开关后，悬浮提示不再出现
- 重新启用后，前缀监听与映射动作恢复生效
- 验证按下 `Caps Lock` 时不再触发系统大小写锁定
- 验证悬浮提示会明确显示 Prefix Pressed / Prefix Released
- 验证悬浮提示默认出现在右侧，并在顶部 / 左侧 / 右侧切换后立即生效
- 在小屏幕或低分辨率下验证悬浮提示不会超出屏幕，左右侧仍能直接完整展示全部说明
- 调整悬浮提示透明度后，验证整个悬浮框会同步变化，包括文字、背景、边框和状态标识
- 在 TextEdit 中验证 `Caps + s` 是否变成左移
- 验证 `Caps + shift + s` 是否变成向左选择
- 验证 `Caps + q` 是否触发 `Control + A`
- 验证命中映射时原始字母不会漏出

### 运行总开关与全局快捷键

- 概览页可直接启用/暂停 Caps Nav
- 概览页可设置、修改、清空全局开关快捷键
- 全局开关快捷键为空时，不应报错
- 全局开关快捷键若无修饰键，应阻止保存并给出提示

### 快捷键练习

- 从菜单栏、设置页概览、App 菜单都能打开练习窗口
- 练习窗口打开时应居中显示，且不会超出窗口范围
- 当前激活配置方案切换后，练习题库会同步切换
- `认键练习` 与 `连招挑战` 都能正常开始、结束并重新开始
- 已授权时，可以按真实 `Caps Lock + 触发键` 完成答题
- 未授权时，可以通过“先按一次 Caps，再输入答案键”的方式完成 fallback 训练
- 答对时对应键位会高亮，答错时会显示正确答案
- 本轮正确率、平均反应时间、最佳连击、易错项会正常更新

### 配置方案编辑

- 激活 `默认方案` 时，按键映射区域应显示只读提示
- 激活 `默认方案` 时，不能新增、修改、删除任意映射
- 复制 `默认方案` 后，新方案应立即可编辑
- 按键映射选择 `builtin` 内置动作后，“功能效果”列应显示预览小图标，悬停后能看到循环动画
- 内置动作预览浮层应支持手动切换“动作前 / 动作后”，并支持暂停/恢复自动播放
- 按键映射选择 `shortcut` 自定义快捷键后，“功能效果”列应显示说明图标，悬停后能看到快捷键说明
- 映射行中的触发键与功能菜单在只读状态下不可点击

### 兼容性

- TextEdit
- Safari 或 Chrome 输入框
- VS Code
- Xcode
- Terminal 或 iTerm2
- 中文输入法候选框激活状态

## 已知风险

- Caps Lock 按下/松开状态在不同机器上可能有差异
- Terminal、IDE、Web 编辑器对 `Option + Arrow` 的解释可能不同
- 中文输入法候选框可能优先消费方向键和删除键
