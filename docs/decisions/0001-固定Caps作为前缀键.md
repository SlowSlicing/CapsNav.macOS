# ADR 0001: Fixed Caps Prefix

## 状态

Accepted

## 决策

Caps Nav 固定使用 `Caps Lock` 作为唯一前缀键，不允许用户在 profile 或 settings 中修改。

## 背景

产品定位是“编辑导航层增强器”，而不是热键平台。固定 prefix 可以保持认知一致性，也能让配置格式、事件链路和文档都明显简化。

## 取舍

### 优点

- 配置模型更简单，不需要 `prefix` 字段。
- 事件处理链路更聚焦，只处理一种前缀键。
- 文档、默认 profile、UI 文案都更稳定。

### 缺点

- 用户不能根据个人习惯改成别的前缀键。
- Caps Lock 作为系统特殊键，实现复杂度更高。

## 为什么仍然选择它

产品价值本身就建立在“Caps 进入导航层”的固定肌肉记忆上。即使实现上更难，也优先保证产品一致性，而不是把系统复杂度转嫁给配置系统和 UI。
