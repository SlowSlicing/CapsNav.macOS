# 发布与打包

这份文档聚焦 **发布者 / 维护者视角** 的打包说明。

如果你只是想从源码运行，或者先生成一个本地体验包，请优先阅读：

- [从源码构建与打包](从源码构建与打包.md)

## 打包模式

Caps Nav 当前提供两种打包模式：

- `dev`
  - 用于本地开发打包或测试分发
- `release`
  - 用于正式发布，包含签名、公证和最终验证

## 目标产物

当前脚本会生成：

- `Universal` 通用版 App
  - 同时包含 `arm64` 与 `x86_64`
- 一个可分发的安装包（`.dmg`）
- 一个对应的 `SHA-256` 摘要文件

最低系统版本仍受工程配置约束，目前为：

- `macOS 13.0+`

## 前提条件

执行打包前，建议先确认：

- 本机已安装 Xcode 和 Command Line Tools（命令行开发工具）
- 当前工程能正常构建
- 可以执行 `xcodebuild`
- 可以执行 `swift`
- 系统可用 `hdiutil`、`ditto`、`shasum`、`osascript`

如果你要执行 `release` 模式，还需要：

- 钥匙串里已安装可用的 `Developer ID Application` 证书（Apple 开发者分发签名证书）
- 已配置 `xcrun notarytool` 的 keychain profile（公证凭据配置）
- 本机可用 `codesign`、`xcrun`、`spctl`

## 一条命令打包

### `dev` 模式

```bash
./tools/package.sh --mode dev
```

脚本会自动完成：

1. 构建 `Release`
2. 生成 `arm64 + x86_64` 通用版 App
3. 校验双架构
4. 生成视觉化安装包（DMG）
5. 生成 `.dmg.sha256`

默认输出目录：

```text
artifacts/dev/
```

### `release` 模式

```bash
./tools/package.sh --mode release \
  --signing-identity "Developer ID Application: Example Corp (TEAM123456)" \
  --team-id "TEAM123456" \
  --notary-profile "capsnav-release"
```

脚本会自动完成：

1. `xcodebuild archive`
2. 生成 `ExportOptions.plist`
3. `xcodebuild -exportArchive`
4. 校验签名后的导出 App
5. 生成最终安装包（DMG）
6. `xcrun notarytool submit --wait`
7. `xcrun stapler staple`
8. `xcrun stapler validate`
9. `spctl` 最终验证
10. 生成 `.dmg.sha256`

## 常用参数

### 自定义输出目录

```bash
./tools/package.sh --mode dev --output-dir /absolute/path/to/output
```

### 自定义 DerivedData 目录

```bash
./tools/package.sh --mode dev --derived-data /absolute/path/to/.build/ReleasePackaging
```

### 跳过 SHA-256 摘要

```bash
./tools/package.sh --mode dev --skip-sha256
```

## `release` 模式的参数来源

`release` 模式必须拿到这三个值：

- `signing identity`
- `team id`
- `notary profile`（公证凭据名称）

脚本读取优先级是：

1. 命令行参数
2. 环境变量
3. 缺失时报错退出

对应环境变量：

```bash
export CAPS_NAV_SIGNING_IDENTITY="Developer ID Application: Example Corp (TEAM123456)"
export CAPS_NAV_TEAM_ID="TEAM123456"
export CAPS_NAV_NOTARY_PROFILE="capsnav-release"
```

然后可以直接执行：

```bash
./tools/package.sh --mode release
```

## `notarytool` 凭据建议

建议先在发布机器上保存一次凭据：

```bash
xcrun notarytool store-credentials "capsnav-release" \
  --apple-id "your-account@example.com" \
  --team-id "TEAM123456" \
  --password "app-specific-password"
```

之后脚本里只传 `--notary-profile` 即可，不要把真实 Apple ID 和密码写进仓库。

## 产物命名

当前产物命名格式为：

```text
Caps-Nav-<版本号>-<构建号>-macOS-universal.dmg
```

例如：

```text
Caps-Nav-1.0-1-macOS-universal.dmg
```

对应摘要文件：

```text
Caps-Nav-1.0-1-macOS-universal.dmg.sha256
```

## 安装包（DMG）内容

生成的安装包默认包含：

- `Caps Nav.app`
- `Applications` 快捷方式

用户打开安装包后，可以直接把 App 拖到“应用程序”目录。

## 视觉化安装盘

当前安装包会额外完成这些处理：

- 自动生成品牌化背景图
- 设置访达（Finder）窗口尺寸
- 固定 `Caps Nav.app` 与 `Applications` 的摆位
- 显示中文拖拽安装提示

相关实现文件：

- `tools/package.sh`
- `tools/GenerateDMGBackground.swift`

## 建议的发布后检查

打包完成后，建议至少手工检查：

1. 安装包（DMG）能正常打开
2. `Caps Nav.app` 和 `Applications` 图标都存在
3. App 拖入“应用程序”后能正常启动
4. 首次启动后菜单栏图标正常出现
5. 权限申请、设置页和前缀键监听都能正常工作

如果你想额外确认架构，可以执行：

```bash
lipo -info '/path/to/Caps Nav.app/Contents/MacOS/Caps Nav'
```

预期输出中应同时出现：

- `arm64`
- `x86_64`

## 对外分发时的说明建议

如果你分发的是 `dev` 模式产物，需要提前告诉用户：

- 第一次打开时，macOS 可能拦截
- 需要到“系统设置 -> 隐私与安全性”中选择“仍要打开”

如果你分发的是 `release` 模式产物，仍建议用户：

- 首次启动后先完成辅助功能权限授权
- 再去测试默认键位和配置方案功能
