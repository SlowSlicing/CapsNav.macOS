# 从源码构建与打包

这份文档面向开发者、测试者和想自己生成安装包的用户。

如果你只是想安装和使用 Caps Nav，优先阅读仓库根目录的 [README.md](../../README.md)。

## 适用场景

这份文档适合你在这些情况下使用：

- 想从源码运行 Caps Nav
- 想在本机调试设置页、权限流程和键位映射
- 想自己打一个本地可分发的安装包（`.dmg`）
- 想确认仓库当前默认构建方式和产物位置

## 环境要求

- macOS 13.0+
- 已安装 Xcode（Apple 官方开发工具）
- 已安装 Xcode Command Line Tools（命令行开发工具）
- 能在终端执行 `xcodebuild`

如果你要生成安装包，还需要本机能正常使用：

- `swift`
- `hdiutil`
- `ditto`
- `shasum`
- `osascript`

## 获取源码

```bash
git clone <your-repo-url>
cd CapsNav.macOS
```

## 方式一：使用 Xcode 运行

这是最适合日常开发和 UI 调试的方式。

步骤：

1. 用 Xcode 打开 `Caps Nav.xcodeproj`
2. 选择 `CapsNav` scheme
3. Destination 选择当前 Mac
4. 直接运行

运行后请注意：

- App 默认只显示菜单栏图标
- 不会自动弹出设置页
- 首次完整体验前仍需要授予辅助功能权限

## 方式二：命令行构建

在仓库根目录执行：

```bash
xcodebuild -project 'Caps Nav.xcodeproj' -scheme 'CapsNav' -configuration Debug -destination 'platform=macOS' -derivedDataPath .build/DerivedData build
```

预期结果：

- 命令结束时出现 `BUILD SUCCEEDED`
- 构建产物位于 `.build/DerivedData/`

## 本地打包一个可分发的安装包（DMG）

如果你想生成一个本地测试包，可以直接使用仓库自带脚本：

```bash
./tools/package.sh --mode dev
```

这里的 `dev` 不是 `Debug` 包，也不是 Xcode 意义上的“开发包”。

它更准确地说是：

- 一个基于 `Release` 配置构建的本地分发包
- 会生成可安装的通用版 App 和安装包（DMG）
- 但不会做 `Developer ID` 正式签名、公证和票据贴附

这个命令会自动完成：

1. 构建 `Release` 版本
2. 生成 `arm64 + x86_64` 通用版 App
3. 生成可双击打开的安装包（DMG）
4. 生成对应的 `SHA-256` 摘要文件

默认输出目录：

```text
artifacts/dev/
```

## 常用变体

### 自定义输出目录

```bash
./tools/package.sh --mode dev --output-dir /absolute/path/to/output
```

### 自定义 DerivedData 目录

```bash
./tools/package.sh --mode dev --derived-data /absolute/path/to/.build/ReleasePackaging
```

### 跳过 SHA-256 摘要

```bash
./tools/package.sh --mode dev --skip-sha256
```

## 产物在哪里

### 命令行构建

默认位于：

```text
.build/DerivedData/
```

### 本地分发打包（未正式签名 / 公证）

默认位于：

```text
artifacts/dev/
```

### 正式发布打包

默认位于：

```text
artifacts/release/
```

## 本地验证建议

在你准备把构建结果发给别人之前，建议至少检查这些内容：

1. App 能正常启动
2. 菜单栏图标能正常出现
3. 设置页能从菜单栏打开
4. 未授权时权限说明窗口能正常出现
5. 授权后 `Caps Lock + 默认键位` 能在任意普通文本编辑器或输入框中生效
6. 如果生成了安装包（DMG），确认拖拽安装路径和窗口布局正常

## 常见问题

### 为什么运行后看不到窗口？

这是正常行为。Caps Nav 默认是菜单栏常驻应用，请从菜单栏图标进入设置页。

### 为什么功能不生效？

通常优先检查：

- 是否已经授予辅助功能权限
- 应用当前是否处于暂停状态
- 是否在终端、IDE 或输入法候选框中测试了某些会被目标应用改写的快捷键

### 为什么脚本里不是 `package-release.sh`？

当前仓库实际脚本文件是：

```text
tools/package.sh
```

如果你在旧文档、历史笔记或缓存命令里看到 `package-release.sh`，请以当前仓库文件名为准。
