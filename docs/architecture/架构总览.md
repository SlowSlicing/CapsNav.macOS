# 架构总览

## 总体结构

Caps Nav 采用“菜单栏入口 + 本地配置存储 + 全局事件链路”三层结构：

1. UI 与系统集成层
2. 配置与状态管理层
3. 键盘事件处理层

## 模块关系

- `CapsNavApp` 负责启动场景。
- `AppBootstrap` 负责组装依赖、初始化目录、加载 settings/profile、刷新权限并启动事件监听。
- `SettingsStore` 与 `ProfileLoader` 负责本地 JSON 读写。
- `ProfileManager` 负责 active profile 选择与切换。
- `PermissionManager` 负责辅助功能权限检查与提示。
- `KeyEventInterceptor` 负责事件 tap、触发器识别和原始按键拦截。
- `PrefixStateManager` 负责维护 Caps prefix 状态。
- `ActionResolver` 负责把 `builtin` 或 `shortcut` 输出转成可发送的 `KeyStroke`。
- `EventEmitter` 负责合成系统键盘事件。

## 关键约束

- Prefix 固定为 `Caps Lock`，不从配置读取。
- App 必须以菜单栏模式运行，所以工程启用了 `LSUIElement = YES`。
- 全局事件拦截依赖 `CGEventTap`，因此目标工程关闭了 App Sandbox。
- 为了避免系统原生 Caps 锁定行为，运行时优先把 `Caps Lock` 重映射为 `F18`，再把 `F18` 作为内部 prefix。

## 当前代码入口

- 启动组装：`Caps Nav/App/AppBootstrap.swift`
- 菜单栏菜单：`Caps Nav/Features/MenuBar/MenuBarMenuView.swift`
- 设置窗口：`Caps Nav/Features/Preferences/PreferencesRootView.swift`
- 事件链路：`Caps Nav/System/`
