# PrefixStateManager

## 模块职责

- 维护 prefix 是否处于 active 状态。
- 只处理 prefix 状态，不直接参与 profile 查找和事件发送。

## 代码入口

- `Caps Nav/System/PrefixStateManager.swift`

## 输入输出

- 输入：来自 `KeyEventInterceptor` 的 `flagsChanged` / Caps 事件。
- 输入：来自 `KeyEventInterceptor` 的 remapped prefix key down/up，或 raw caps fallback 事件。
- 输出：`isPrefixActive` 状态。

## 依赖关系

- 被 `KeyEventInterceptor` 调用。
- 不依赖 UI、配置存储和事件发送。

## 典型调用路径

常规路径：

`KeyEventInterceptor.handleKeyDown(_:)`  
-> `PrefixStateManager.handlePrefixKeyDown()`

`KeyEventInterceptor.handleKeyUp(_:)`  
-> `PrefixStateManager.handlePrefixKeyUp()`

## 边界条件

- Raw Caps fallback 仍然不如 remapped F18 稳定，因此只作为降级路径。
- 当事件 tap 停止时，需要主动 `reset()`，避免脏状态遗留。
