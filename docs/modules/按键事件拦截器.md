# KeyEventInterceptor

## 模块职责

- 创建并维护全局 `CGEventTap`。
- 监听 `keyDown`、`keyUp`、`flagsChanged`。
- 在 prefix active 期间识别 trigger，并优先处理 remapped prefix key。
- 命中映射时拦截原始事件，并转发到 resolver/emitter。

## 代码入口

- `CapsNav/System/KeyEventInterceptor.swift`

## 依赖

- `PermissionManager`
- `PrefixKeyRouter`
- `PrefixStateManager`
- `ActionResolver`
- `EventEmitter`
- active profile provider

## 典型调用路径

`AppBootstrap.refreshAccessibilityStatus()`  
-> `PrefixKeyRouter.activatePreferredRouting()`  
-> `KeyEventInterceptor.start()`  
-> `CGEvent.tapCreate(...)`  
-> `handleFlagsChanged / handleKeyDown / handleKeyUp`

## 边界条件

- 没有辅助功能权限时不能启动 tap，也不应该保留 Caps remap。
- App Sandbox 必须关闭，否则事件拦截不可行。
- `tapDisabledByTimeout` / `tapDisabledByUserInput` 需要自动重新 enable。

## 当前实现说明

- 当前已经完成全局 tap 骨架、Caps -> F18 路由和 trigger 命中链路。
- 当前通过 `keyboardGetUnicodeString` 解析字母键，适合 v1 默认映射，后续可视需要补充更稳定的物理键码策略。
